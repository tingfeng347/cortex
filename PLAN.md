# 认知防锈 (Cognitive Anti-Rust)

## 项目定位

一个认知健康追踪工具，帮助用户感知、测量、延缓因过度依赖 AI 而导致的核心心智能力退化。

核心假设：**当人们看见自己的某项脑力正在因 AI 使用而退化时，他们会在意，并且会分享。**

这个假设是整个项目的 gate——Phase 0 就是要验证它。

目标：让用户有理由回来，形成复测习惯 ✅

这一阶段不做复杂算法，专注于行为设计：

4. **数据洞察** ✅

5. **其他 UI / 基建** ✅

   - Error Boundary（全局错误捕获 + 重试按钮）

### Phase 1：动态评估引擎

时长：2-4 周
需要 Agent：单 Agent

- 基于 IRT 或贝叶斯更新的自适应出题
- LLM 动态生成题干和干扰项
- 用户能力画像（多维度：逻辑、速算、语义独立评分）
- 可选的深度评估模式（更多题目、更细粒度）

### Phase 2：微干预引擎

时长：1-2 月
需要 Agent：双 Agent（评估 + 教练）

- 评估 Agent：持续评估退化曲线
- 教练 Agent：设计微型训练计划
- 两个 Agent 共享用户画像但以不同方式使用
- 必要时需要协商（测量精度 vs 用户倦怠）

### Phase 3：认知镜像与盲区揭示

时长：3-6 月
需要 Agent：成熟 Multi-Agent

- 观察 Agent：分析用户授权提供的文本/决策痕迹
- 建模 Agent：跨技能域认知图谱
- 呈现 Agent：用不冒犯的方式揭示盲区
- 隐私安全 Agent：全本地/加密处理

**这是整条路线最有力量的洞察**——用户不知道自己不知道在退化什么。
但隐私基建不是 Phase 3 才该想的，Phase 0 就应该建立"全部本地处理"的信任。

### Phase 4：认知防锈经济

时长：6 月+
需要 Agent：Multi-Agent + 市场机制

雇主匹配 Agent、保险定价 Agent、去中心化认知信用记录。

## 关键架构原则

### 隐私优先

每个阶段的设计都必须向后兼容本地优先的隐私承诺。
如果 Phase 0-2 上传数据，Phase 3 突然要求本地分析，用户不会信任。

当前匿名策略：
- 不存任何用户标识（无 cookie、无 fingerprint）
- 每个结果是匿名的
- 数据仅用于生成统计分布

### 关于 Agent 架构

- 这个路线图的 Agent 不是营销概念，是不得已而为之
- Phase 2 开始需要 Multi-Agent，因为"评估"和"干预"是天然冲突的两个目标
- 与 MiroFish（580 Agent 模拟红楼梦）的关键区别：
  MiroFish 的 Agent 不需要对齐到同一个外部真相；
  而认知防锈的评估 Agent 和教练 Agent 必须对同一个用户的能力状态达成一致理解。
  这意味着需要一个比 message bus 更强的**共享心智模型层**。

### Phase 0 不做 Agent 的原因

- 固定题库 + 规则评分 = 结果完全可解释、可复现
- 用户拿到的退化指数是确定的，不是 AI 黑箱
- 这对信任敏感的场景（告诉用户他的大脑在变差）至关重要
